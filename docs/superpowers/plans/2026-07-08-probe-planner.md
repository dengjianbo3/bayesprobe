# Probe Planner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a deterministic Probe Planner that selects a bounded, cycle-frozen `ProbeSet` from a `ProbeCandidate` pool and current `BeliefState`.

**Architecture:** Add `bayesprobe/probe_planner.py` as a focused planning module. It ranks and freezes probes only; it does not execute tools, create `ExternalSignal`s, interpret evidence, update beliefs, or evolve hypotheses. The planner consumes `ProbeCandidate`s generated by initialization and returns a `ProbeSet` that existing core/controller code can consume later.

**Tech Stack:** Python 3.11+, dataclasses, existing Pydantic schemas in `bayesprobe.schemas`, existing `JsonlLedgerStore`, pytest.

## Global Constraints

- `ProbeCandidate` is not an action, tool result, or evidence.
- `ProbeSet` is the bounded active control signal frozen for one cycle.
- The planner must not execute probes.
- The planner must not create `ExternalSignal`s.
- The planner must not interpret evidence or update posterior beliefs.
- The planner must not evolve hypotheses.
- Do not implement LLM scoring, ToolGateway, benchmark scoring, or cross-cycle candidate lifecycle management in this slice.
- Do not attempt git commits because this workspace is not currently a git repository.

### Task 2: Probe Planner Module

**Files:**
- Create: `bayesprobe/probe_planner.py`

**Interfaces:**
- Consumes: `BeliefState`, `ProbeCandidate`, `ProbeDesign`, `ProbeSet`, `JsonlLedgerStore`
- Produces:
  - `ProbePlanningConfig`
  - `RejectedProbeCandidate`
  - `ProbePlanningResult`
  - `ProbePlanner.design_probe_set(...) -> ProbePlanningResult`

- [x] **Step 1: Add public dataclasses**

Implement:

```python
@dataclass(frozen=True)
class ProbePlanningConfig:
    max_probes: int = 2
    allow_empty: bool = False
    attack_top_hypothesis_bonus: float = 1.25
    unresolved_uncertainty_bonus: float = 1.1
```

with validation for positive `max_probes` and positive bonuses.

Implement:

```python
@dataclass(frozen=True)
class RejectedProbeCandidate:
    candidate: ProbeCandidate
    reason: str
    score: float


@dataclass(frozen=True)
class ProbePlanningResult:
    probe_set: ProbeSet
    selected_candidates: list[ProbeCandidate]
    rejected_candidates: list[RejectedProbeCandidate]
```

- [x] **Step 2: Implement deterministic ranking and freezing**

`ProbePlanner.design_probe_set(...)` should:

1. Validate `run_id`, `cycle_id`, and config.
2. Reject invalid candidates with reasons:
   - `invalid_no_targets`
   - `invalid_unknown_targets`
3. Score valid candidates:

```text
score =
  expected_information_gain
  * decision_relevance
  * attack_top_hypothesis_bonus_if_probe_targets_top
  * unresolved_uncertainty_bonus_if_belief_state_has_uncertainty
  / max(cost_estimate, 0.01)
```

4. Sort by higher score, lower cost, candidate ID ascending.
5. Ensure at least one selected probe attacks the top hypothesis when such a valid candidate exists.
6. Freeze selected probes to `cycle_id`.
7. Mark selected candidates with `selected_in_cycle=cycle_id`.
8. Return budget-limit rejections for valid candidates not selected.

- [x] **Step 3: Implement empty behavior and ledger write**

When no valid candidates remain:

- `allow_empty=False`: raise `ValueError`.
- `allow_empty=True`: return empty `ProbeSet(may_be_empty=True)`.

If a ledger is provided, append exactly one `probe_set` record after successful planning.
